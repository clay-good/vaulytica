## Deviations

- **Citation grammar lives in `src/extract/citation-grammar.ts` as cited data**, not as DKB grammar nodes with a build/floor-check (same rationale as the filing pack's court profiles: bundles for the browser and imports for the CLI with no runtime fetch/fs; deterministic everywhere). It carries `INDIGO_BOOK_SOURCE` (cite/url/retrieved_at) and the reporter table; cited as "The Indigo Book 2.0" — never "Bluebook".
- **CITE rules activate on any filing brief, independent of `--court`.** Citation format/consistency needs no court profile, so `activateFiling` adds `CITE_RULES` whenever the matched playbook is a filing playbook, and `FILING_RULES` (format) only additionally when `--court` is set. Non-filing documents never match a filing playbook, so all existing hashes are unchanged (isolation preserved).
- **One aggregate finding per rule.** The engine's `check` returns a single `Finding`, so each CITE rule fires once, reporting the count and up to five examples, anchored at the first offending instance — matching the STRUCT-rule convention rather than one finding per citation.
- **CITE-005 and short-form matching are conservative heuristics.** The extractor captures case citations as volume/reporter/page (no party name), so "introduction" of an authority is inferred from an adjacent `X v. Y` short-case within 60 flat chars of a full citation. This can miss exotic forms; it errs toward silence (no false alarms) rather than over-flagging.
- **TOA reconciliation is by authority, not page number** (v1 scope, as the proposal states) — the flattened tree has no page boundaries.
